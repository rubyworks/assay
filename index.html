Build Solid Assertion Frameworks

RESEARCH STUDY #A-0-4-0

Hypothesis: All know Ruby test frameworks utilize a subclass of Exception to represent failed assertion. We postulate that all assertions can be most effectively modeled by exception subclasses.

Apparatus: Project defines set of reusable assertion classes mimicking the design of Ruby exception hierarchy. The Assertion class serves as base class for all other assays and is subclass of Exception. A complete set of assay assertion classes are defined for all common assertion needs.

Implementation Note: Standard design is based on BRASS project, allowing use with any compliant test system. Included adapters provide integration with common legacy test frameworks: TestUnit, MiniTest and RSpec.

Prime Corollary: Third-party assertion grammars can use Assay as solid foundation for own assertion domains.

Details of Study available at API and QED documentation sites.
